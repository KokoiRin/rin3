# 协作约定

- 默认简短回答：先讲结论和最关键判断，重要内容讲清楚，不主动展开不必要的背景、例子或延伸分析。
- 验证按改动风险分级：纯内部重构且不改变行为时，默认只运行针对性测试、类型检查和 lint；不要重复运行多套等价的完整构建。
- 只有本轮修改了 UI 交互、正在修复交互回归、用户明确要求，或浏览器验证能覆盖自动化测试无法覆盖的明确风险时，才运行浏览器测试。不要仅为普通重构临时下载或引入新的浏览器测试基础设施；确有长期价值时，先说明额外成本并征得用户同意。
- 每次修改代码或内容后，会话结束前提供一个已经验证可访问的本地预览链接，方便用户立即验收。
- 如果本轮修改已经部署到 GitHub Pages 或其他远程网页，同时提供对应的远程网页链接；不要只提供 GitHub 仓库链接。
